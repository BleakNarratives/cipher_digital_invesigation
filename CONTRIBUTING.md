## Quick Start

1. Fork the repo
2. Create a branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Test thoroughly
5. Commit: `git commit -m "Add: your feature"`
6. Push: `git push origin feature/your-feature`
7. Open a Pull Request

## What We're Looking For

### High Priority
- Bug fixes
- Performance improvements
- Better error messages
- UI/UX improvements
- Browser compatibility fixes
- Documentation improvements

### Medium Priority
- New recovery algorithms
- Additional blockchain support
- Accessibility improvements
- Internationalization (i18n)

### Low Priority (Validate First!)
- New features (must solve real user problems)
- Major refactors (discuss first)

## Guidelines

### Code Style
- Use clear, descriptive variable names
- Add comments for complex logic
- Keep functions small and focused
- Follow existing code patterns

### Testing
- Test in multiple browsers (Chrome, Firefox, Edge, Safari)
- Test on mobile devices
- Test with real mnemonics (use test wallets!)
- Verify privacy (no network calls except CDN)

### Commits
- Use clear commit messages
- One logical change per commit
- Reference issues when applicable

### Pull Requests
- Describe what and why
- Include screenshots for UI changes
- Link to related issues
- Keep PRs focused and small

## Reporting Bugs

Open an issue with:
- Browser and version
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Console errors if any

## Feature Requests

Before requesting features:
1. Check if it already exists
2. Explain the problem it solves
3. Describe your use case
4. Consider if it fits the MVP philosophy

## Philosophy

This project follows lean startup principles:
- **Start small** - One feature done well
- **Ship fast** - Iterate based on feedback
- **User value** - Solve real problems
- **Keep it simple** - Complexity is the enemy

Read `docs/TRANSFORMATION.md` to understand our approach.
